1. OVERVIEW
This runbook covers day-to-day operations for the sharded PostgreSQL cluster
introduced during Project Atlas (see Project Atlas Postmortem for migration
history).

2. ROUTINE OPERATIONS
- Shard rebalancing: run monthly, script located at ops/rebalance_shards.sh
- Backup verification: automated, alerts sent to #infra-alerts if a backup
  fails checksum validation.
- Failover testing: quarterly, requires manual coordination with on-call.

3. KNOWN ISSUES
- Shard 4 occasionally shows replication lag under high write load. Root
  cause not fully understood; current mitigation is a manual restart of the
  replication worker. Deepak Rao is currently the only person who knows the
  exact restart sequence and has not yet documented the precise steps.
- Connection pool exhaustion has occurred twice (Dec 2024, Jan 2025) under
  traffic spikes. Deepak Rao resolved both manually via direct DB access;
  no automated alert exists for this yet.

4. NOTE FOR FUTURE MAINTAINERS
Most of the tribal knowledge for this cluster currently exists only with
Deepak Rao. If Deepak is unavailable, escalate to the original migration
team (Karthik Subramanian) who has partial context but was not involved in
day-2 operations setup.
